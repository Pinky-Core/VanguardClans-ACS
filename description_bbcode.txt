[SIZE=5][B]Installation[/B][/SIZE]
[LIST=1]
[ *]Download the latest build of VanguardClans.
[ *]Drop [ICODE]VanguardClans.jar[/ICODE] inside your [ICODE]/plugins[/ICODE] folder.
[ *]Edit [ICODE]config.yml[/ICODE] to pick storage (YAML/SQLite/MariaDB), nametag provider (internal/TAB/UnlimitedNametags), webhook targets, and anti-multiaccount limits.
[ *]Restart your server or run [ICODE]/clanadmin reload[/ICODE] and let the plugin generate defaults.
[ *]Optional: install PlaceholderAPI, TAB/UnlimitedNametags, and choose your language with [ICODE]/clanadmin lang select <code>[/ICODE].
[/LIST]

[SIZE=5][B]Features[/B][/SIZE]
[LIST]
[ *]Public/private clans with hierarchical roles, slots, friendly fire toggles, teleport homes, and auto-cleanup when empty.
[ *]Alliance system with requests, accept/decline/removal, friendly fire control, and invite auto-expiry every 5 minutes.
[ *]Slot upgrades fueled by points, kills, and configurable costs plus `/clan slots` overview and buy wizard.
[ *]Dynamic GUIs for members, allies, invites, and tops showing KDA, total kills/deaths, points, money, and member count.
[ *]Leaderboard placeholders `%vanguardclans_top_<metric>_<position>[_<field]%` for KDA, points, money, members, kills, deaths, and combined stats.
[ *]Nametag privacy that hooks into the internal scoreboard, TAB, or UnlimitedNametags and supports obfuscation, hide, or reset modes.
[ *]Anti-multiaccount safeguards that can limit clan creation per IP whenever the global `max-clans` cap is reached.
[ *]Discord webhooks for new clan creation and leaderboard shifts, fully toggleable in `config.yml` with separate channels.
[ *]Multi-language engine with English/Spanish defaults, easy-to-edit `[lang/<code>.yml]`, and `/clanadmin lang select`.
[ *]Storage-agnostic: YAML, SQLite, MariaDB with Hikari, full migrations, cache refresh, and stats/history APIs.
[ *]Lightweight, modular architecture built for high concurrency while leaving room for subsection hooks (TAB, nametags, unlimited, webhooks).
[/LIST]
